📝 Story Agent — System Prompt
You are a senior Product Owner and Business Analyst with 10+ years experience.
Your job: receive a Jira Epic and decompose it into well-structured user stories.

For EACH story you MUST output:
1. Story Title (verb-noun format: "Create User Login Screen")
2. User Story (As a [role], I want [action], So that [benefit])
3. Acceptance Criteria (Given/When/Then, minimum 3 criteria)
4. Story Point Estimation using Fibonacci (1,2,3,5,8,13)
   - Justify points based on: complexity, uncertainty, effort
5. Dependencies (which other stories must complete first)
6. Suggested Microservice owner

Story Point Reference:
1pt = trivial UI change | 2pt = simple CRUD | 3pt = standard feature
5pt = complex logic/integration | 8pt = multi-service feature | 13pt = epic-level

Always ask clarifying questions if the epic is ambiguous.
Output in structured markdown that can be pasted into Jira.
🏛 Architect Agent — System Prompt
You are a Principal Solution Architect specializing in cloud-native microservices.
Tech stack: Java 17, Spring Boot 3.x, Spring Security, JPA/Hibernate, PostgreSQL,
React 18 (TypeScript), Docker, Kubernetes (GKE), REST APIs.

For each architecture task output:
1. Service Boundary Analysis (what belongs where)
2. API Contract (OpenAPI 3.0 spec excerpt)
3. Data Model (entity diagrams in text/mermaid)
4. Inter-service Communication pattern (REST/event-driven)
5. Security design (JWT, RBAC)
6. Non-functional considerations (scaling, resilience)

Always follow 12-factor app principles.
Always consider single responsibility per service.
Flag any risks or anti-patterns you see in the requirements.
💻 Dev Agent — Claude Code CLAUDE.md

Create a CLAUDE.md file in your project root. Claude Code reads this automatically.

# CLAUDE.md — Dev Agent Instructions

## Stack
- Java 17, Spring Boot 3.2, Maven
- Spring Data JPA, PostgreSQL
- Spring Security 6 with JWT
- React 18, TypeScript, Axios
- JUnit 5, Mockito, TestContainers

## Coding Standards
- Follow SOLID principles
- Every public method needs Javadoc
- Service layer must be unit-tested (>80% coverage)
- Use constructor injection (no @Autowired on fields)
- All REST endpoints must return ResponseEntity<>
- Use DTOs — never expose JPA entities directly
- Validate inputs with Bean Validation (@Valid)

## File Structure
/services/user-service     → User Management Service
/services/order-service    → Order Management Service
/frontend/ui-app           → React frontend

## Before any commit
- mvn test must pass
- No TODO comments left in code
- Run: mvn spotless:check

Phase 1 — Epic to User Stories
Story Agent · Claude.ai Projects · ~30 min
Input: Your Jira Epic

Example Epic for our sample application:

EPIC: E-Commerce Platform — Core Foundation
As a business, we need a web application where:
- Admins can manage users (create, edit, deactivate, assign roles)
- Customers can register, login, and manage their profile  
- Customers can browse, create, and manage orders
- The system must support multiple user roles: ADMIN, MANAGER, CUSTOMER
- All services communicate via REST APIs
- Deployed on Google Cloud Kubernetes Engine
How to Use the Story Agent
  1. 1
    Open your "📝 Story Agent" Claude Project

    The system prompt is already loaded. Paste your Epic text into the chat.

  2. 2
    Say: "Break this epic into user stories grouped by microservice"

    Claude will output structured stories with story points for each service.

  3. 3
    Review and iterate

    Ask: "Add acceptance criteria for the RBAC story" or "Split story 3 — it's too large"

  4. 4
    Export to Jira

    Ask: "Format all stories as Jira-compatible markdown with labels and components"

Expected Story Output (Example)
## STORY: US-001 — User Login
As a registered user, I want to log in with email/password
so that I can access the platform securely.

Acceptance Criteria:
- Given valid credentials → JWT token returned, redirect to dashboard
- Given invalid credentials → 401 error with message "Invalid credentials"  
- Given account locked → 403 with message "Account locked"
- Token expires after 24 hours
- Remember-me option extends to 7 days

Story Points: 3
Dependencies: US-002 (Register User)
Service: user-service, ui-app

## STORY: US-007 — Admin User Management Dashboard
Story Points: 8  (complex UI + role-based API + pagination)
Dependencies: US-001, US-002, US-003

User Service — Data Model
@Entity
User {
  id: UUID
  email: String (unique)
  passwordHash: String
  firstName: String
  lastName: String
  role: Enum(ADMIN,MANAGER,CUSTOMER)
  status: Enum(ACTIVE,INACTIVE,LOCKED)
  createdAt: Timestamp
  updatedAt: Timestamp
}

RefreshToken {
  id: UUID
  user: User (FK)
  token: String
  expiresAt: Timestamp
}
Order Service — Data Model
@Entity
Order {
  id: UUID
  userId: UUID (ref, no FK)
  status: Enum(PENDING,CONFIRMED,
         SHIPPED,DELIVERED,CANCELLED)
  totalAmount: BigDecimal
  createdAt: Timestamp
  updatedAt: Timestamp
}

OrderItem {
  id: UUID
  order: Order (FK)
  productName: String
  quantity: Integer
  unitPrice: BigDecimal
}

Key Claude Code Prompts Per Service

USER SERVICE

"In user-service, implement:
1. User entity with fields from ARCHITECTURE.md
2. UserRepository (JPA)
3. UserService with register/login/CRUD
4. AuthController POST /auth/register, /auth/login
5. UserController GET/PUT/DELETE /users/**
6. JWT filter chain in SecurityConfig
7. Role-based access (ADMIN only for DELETE)
8. DTOs: RegisterRequest, LoginResponse, UserDTO
Use constructor injection throughout."

ORDER SERVICE

"In order-service, implement:
1. Order + OrderItem entities
2. OrderRepository, OrderItemRepository
3. OrderService with create/update/cancel
4. OrderController with endpoints:
   POST /orders
   GET /orders/{id}
   GET /orders/user/{userId}
   PUT /orders/{id}/status
   DELETE /orders/{id}
5. JWT validation filter (verify token from user-service)
6. Status state machine (PENDING→CONFIRMED→SHIPPED)"
React Frontend Key Prompts
"In /frontend, create these React components:
1. AuthContext — stores JWT, expiry, user role
2. PrivateRoute — redirects to login if no token
3. LoginPage — form with email/password, calls POST /auth/login
4. Dashboard — role-based nav (ADMIN sees User Management, all see Orders)
5. UserManagementPage (ADMIN only):
   - Table of all users with search/filter
   - Modal to edit user details and role
   - Toggle active/inactive status
6. OrdersPage:
   - List user's orders with status badges
   - Create Order modal with items
   - Order detail view with timeline
Use Axios with interceptor to attach Bearer token automatically."
Claude Code Workflow Tips
  • Review before accepting — Claude Code shows diffs; press y to accept each file
  • Incremental builds — after each feature, run mvn test before moving on
  • Fix errors in-context — paste the error back into Claude Code: "Fix this compilation error: [paste]"
  • Use /compact flag — if context gets long, type /compact to summarize and continue

"Write @SpringBootTest integration tests
using TestContainers PostgreSQL:
- Full auth flow: register → login → get token
- Admin creates user, customer login, view orders
- Order lifecycle: create → confirm → ship → deliver
- Unauthorized access returns 403
- JWT expired returns 401

Use MockMvc for HTTP assertions
Use @Sql to seed test data"

Phase 6 — Docker & GKE Deployment
DevOps Agent · Docker + Kubernetes + Google Cloud
Dockerfile — user-service
# Multi-stage build
FROM maven:3.9-eclipse-temurin-17 AS build
WORKDIR /app
COPY pom.xml .
RUN mvn dependency:go-offline
COPY src ./src
RUN mvn package -DskipTests

FROM eclipse-temurin:17-jre-alpine
RUN addgroup -S app && adduser -S app -G app
WORKDIR /app
COPY --from=build /app/target/*.jar app.jar
USER app
EXPOSE 8081
ENTRYPOINT ["java","-jar","app.jar"]
Kubernetes Deployment — user-service
apiVersion: apps/v1
kind: Deployment
metadata:
  name: user-service
  namespace: ecommerce
spec:
  replicas: 2
  selector:
    matchLabels: {app: user-service}
  template:
    metadata:
      labels: {app: user-service}
    spec:
      containers:
      - name: user-service
        image: gcr.io/PROJECT_ID/user-service:latest
        ports: [{containerPort: 8081}]
        env:
        - name: DB_URL
          valueFrom:
            secretKeyRef:
              name: db-secret
              key: user-db-url
        resources:
          requests: {memory: 256Mi, cpu: 250m}
          limits: {memory: 512Mi, cpu: 500m}
GKE Deployment — Step by Step
  1. 1
    Create GKE Cluster 
    gcloud container clusters create ecommerce-cluster \
  --zone us-central1-a \
  --num-nodes 3 \
  --machine-type e2-standard-2
gcloud container clusters get-credentials ecommerce-cluster --zone us-central1-a
  2. 2
    Build & Push Images to Google Artifact Registry 
    gcloud auth configure-docker us-central1-docker.pkg.dev
docker build -t us-central1-docker.pkg.dev/PROJECT_ID/ecommerce/user-service:v1 \
  ./user-service
docker push us-central1-docker.pkg.dev/PROJECT_ID/ecommerce/user-service:v1
  3. 3
    Create Kubernetes Namespace & Secrets 
    kubectl create namespace ecommerce
kubectl create secret generic db-secret \
  --from-literal=user-db-url="jdbc:postgresql://..." \
  --from-literal=order-db-url="jdbc:postgresql://..." \
  -n ecommerce
  4. 4
    Apply Kubernetes Manifests 
    kubectl apply -f k8s/namespace.yaml
kubectl apply -f k8s/user-service/
kubectl apply -f k8s/order-service/
kubectl apply -f k8s/frontend/
kubectl apply -f k8s/ingress.yaml
  5. 5
    Verify Deployment 
    kubectl get pods -n ecommerce
kubectl get services -n ecommerce
kubectl logs -f deployment/user-service -n ecommerce
