Why Our SDLC Needs to Evolve
Six pain points every developer on the team recognises
⏱ Spike Before Every Story

1–3 days reading legacy code before writing a story. Multiply by 15+ services and 4 sprints a year.

📄 Zero Documentation

5–8 year-old microservices with no docs, no architecture records. Every new developer starts from zero.

🔁 Manual Every Step

Epic → Stories → Design → Code → Tests → Deploy. Each handoff is a context switch, a meeting, a delay.

❌ No Contract Safety

Customer APIs break silently. No automated contract tests. A routine refactor becomes a production incident.

🔍 Questions Slow Work

"Who owns this table?" "Is it safe to change this?" — each question costs hours before work can begin.

🚀 Deployment Anxiety

No deployment plan, no rollback playbook, no smoke test automation. Every GKE deploy is a manual risk.

Phase 3 — Four Agents for Legacy Code
Eliminates the spike story entirely. Scan → Document → Ask → Fix → Generate Stories. All in one session.
⚠️
The Reality: Legacy services have years of tribal knowledge locked in undocumented Java. Phase 3 gives you institutional memory before any change — and eliminates spike stories entirely.
🔍 Code Archaeology Copilot + Claude

Scans the entire repo. Writes 17+ small focused files — one concern per file, under 80 lines. Works in Copilot Chat (@workspace) and Claude Code.

  • SERVICE_PROFILE.md — what this service is
  • auth.md — Apigee + JWT model
  • gke.md — K8s config snapshot
  • deps.md — inter-service blast radius
  • tests.md — JUnit version, coverage state
📝 Story Generator Eliminates Spike Stories

Reads archaeology docs and generates stories directly from CODE — no spike story needed. Type A (existing) + Type B (gaps). Run from Copilot /create-story or Claude story-helper.

  • Type A: existing endpoints → stories
  • Type B: P1 security gaps, P2 coverage
  • Fibonacci estimates from code complexity
  • Jira-ready with auth labels + risk level
💬 Repo QA Agent In Your IDE

Answers any question from context docs — no re-scanning. Ask from Copilot Chat or Claude. Inside your IDE. Your code never leaves your workspace.

  • 'Is it safe to change X?' → risk level
  • 'How does auth work?' → Pattern A/B
  • 'How do I write a test?' → JUnit pattern
  • 'Story points for X?' → estimate
🔧 CodeFixes Folder Exact Line + Code

One file per risky finding. Exact file path, line number, current code block, suggested replacement, and why it's safe. Never auto-applies — developer reviews each.

  • FIX-001: Missing @Valid (P1 Security)
  • FIX-002: JVM OOM risk (P1 GKE)
  • FIX-003: No HTTP timeout (P2)
  • _INDEX.md: all findings by severity

Spike Story Elimination — Before vs After
The workflow your team runs today — replaced entirely
Before — Manual Spike (2–3 days)
  • Dev assigned spike story
  • 2 days reading 50k lines of code
  • Ask 3 engineers for context
  • Write rough Confluence notes
  • PO writes stories from the notes
  • Stories miss edge cases + auth risks
  • Sprint estimate is a guess
  • Total: 2–3 days, still incomplete
After — Code Archaeology + Story Generator (30 min)
  • Open service in VS Code or terminal
  • Run Code Archaeology (15 min, read-only)
  • 17+ context files written for both tools
  • Ask Repo QA: 'what does this service do?'
  • Run Story Generator: stories from docs
  • Stories include auth model, risk, estimates
  • Jira-ready with Type A + Type B split
  • Total: 30 min, production-accurate

Time saved per sprint: ~2 days per service → ~40 dev-days saved per year for a 10-service team.
