# Code Archaeology Skill

Writes **many small focused files** across tool-native locations so both
GitHub Copilot and Claude Code automatically load the right context on every
session — without any manual prompting by the developer. Also generates a
`CodeFixes/` folder: one file per risky/incorrect code finding, with exact
file path, line number, current signature, and suggested replacement code.

---

**Size target**: Every context/agent file under 80 lines. Each CodeFix file under 60 lines.

## Step 1 — Detect Repo Type

Read in order, stop at first match per type:

| File Found | Repo Type | Scan Path |
|------------|-----------|-----------|
| `pom.xml` | Spring Boot / Java | ? [Java Scan](#java-scan) |
| `package.json` + `src/` with `.tsx`/`.jsx` | React UI | ? [React Scan](#react-scan) |
| `package.json` (no React) | Node service | ? [React Scan](#react-scan) |
| Both `pom.xml` + `package.json` | Hybrid/monorepo | ? Run both scans |
| `*.jar` referenced in other pom.xml | Shared JAR | ? [JAR Scan](#jar-scan) |

---

## Step 2 — Java Scan

Read in this sequence. Extract only what's listed — nothing more.

1. `pom.xml` ? `artifactId`, Java version, Spring Boot version, dependency list (name+version only)
2. `src/main/resources/application*.yml` ? `server.port`, datasource URL pattern (redact password), active profiles, feature flags
3. All `@RestController` classes ? HTTP method, path, handler method name, auth annotation
4. `SecurityConfig` or `WebSecurityConfigurerAdapter` ? `permitAll()` paths, filter chain
5. `JwtAuthFilter` (or equivalent) ? header name read, claims extracted, MDC fields set
6. All `@Entity` classes ? class name, table name, key fields (no getters), relationships
7. `src/main/resources/db/migration/` ? count scripts, latest version number
8. All `RestTemplate`/`WebClient`/`FeignClient` beans ? target URL config key, timeout setting
9. `@RabbitListener`/`@KafkaListener`/`@PubSubListener` ? topic/queue names
10. `k8s/*.yaml` or `deploy/*.yaml` ? memory limit, CPU limit, env var names, probe paths
11. `Dockerfile` ? base image, JVM flags set, exposed port
12. `src/test/` ? count test classes vs prod classes, JUnit version (4 or 5), `@Ignore`/`@Disabled` count

---

## Step 3 — React Scan

1. `package.json` ? name, version, key deps (React, router, state mgmt, API client)
2. Router config ? all route paths + page component names
3. Auth logic ? localStorage/sessionStorage key names, token field names, AuthContext shape
4. API client files (axios instance, fetch wrapper) ? base URL config key, auth header injection point
5. `src/components` or `src/pages` ? 2-level component tree (pages ? direct children only)
6. `.env.example` / `.env.sample` ? all key names (never values)
7. `nginx.conf` or `Dockerfile` ? served port, build output directory

---

## Step 4 — JAR Scan

1. `pom.xml` ? groupId, artifactId, current version
2. All public classes in `src/main/java` ? public method signatures only
3. Any consumers in sibling `pom.xml` files ? which services depend on this JAR + which version

---

```markdown
# patterns.md — {service-name}
> Existing patterns. New code MUST match these. Do not invent new patterns.

## Exception Handling
- Handler: GlobalExceptionHandler extends ResponseEntityExceptionHandler
- Response format: `{"code":"ERR_XXX","message":"...","timestamp":"..."}`
- Custom exceptions extend: {BaseException class name}

## DTO Rules
- Always separate: {Entity}Request (in) / {Entity}Response (out)
- Mapping: MapStruct (NOT manual setters or ModelMapper)
- Never expose @Entity objects directly in controllers

## Logging
- Framework: SLF4J + Logback
- MDC fields: requestId, userId, sessionId (set by JwtAuthFilter or interceptor)
- Pattern: `log.info("{} {}", label, value)` — never string concat in log calls
- PII fields (email, phone, name): never log in plain text

## Validation
- On DTOs: @Valid on @RequestBody in controller
- Constraint annotations on DTO fields (not entity fields)
- Errors handled by GlobalExceptionHandler automatically

## Transactions
- @Transactional on Service layer ONLY
- Never on Controller or Repository layer

## Testing (see tests.md for full details)
- JUnit {4/5} — {confirm from pom.xml — do NOT mix versions}
- Base class: {BaseServiceTest / BaseControllerTest} — always extend it
- Mocking: Mockito @Mock + @InjectMocks (unit) / @MockBean (integration)
```

### File 4: `.claude/context/auth.md`

```markdown
# auth.md — {service-name}
> Authentication model. Critical for all API work.

## Gateway: Apigee API Gateway
All external APIs pass through Apigee before reaching this service.

## Pattern A — Basic Auth (most customer APIs)
```
Customer ? [Basic Auth] ? Apigee validates ? strips header ? Service
Service receives: X-Consumer-ID: {customer-id}  (NOT Authorization header)
```
Rule: Service code NEVER parses Basic Auth. Read X-Consumer-ID for identity.

## Pattern B — JWT Passthrough (admin / partner APIs)
```
Customer ? [Bearer JWT] ? Apigee validates signature ? forwards JWT ? Service
Service receives: Authorization: Bearer {token}
```
Rule: JwtAuthFilter validates locally. Claims: {userId, roles[], sessionId, exp}
MDC: sets requestId, userId, sessionId on every request.

## Public Endpoints (no auth)
{List from SecurityConfig permitAll() or @PublicEndpoint scan}
- /actuator/health
- /actuator/health/liveness
- /actuator/health/readiness
- {others found}

## Auth Endpoint Map
| Endpoint Pattern | Pattern | Header Service Receives |
|-----------------|---------|------------------------|
| /api/v1/...     | A or B  | X-Consumer-ID or Bearer |

## Security Rules
- NEVER add .basic() or BasicAuthenticationFilter to SecurityConfig
- NEVER log X-Consumer-ID or JWT token values (use MDC only)
- NEVER disable JwtAuthFilter for non-public endpoints
- CORS: no wildcard (*) origins in production config
```

## Step 5b — Generate CodeFixes

While scanning, collect every finding where code is **incorrect, risky, or
could break functionality if changed carelessly**. Write ONE file per finding
into `CodeFixes/`. This is separate from documentation — these are actionable
fix suggestions with exact code.

### What qualifies as a CodeFix

| Category | Examples |
|----------|----------|
| Security | Missing `@Valid`, Basic Auth parsed in service, secret hardcoded, PII logged |
| Reliability | HTTP call with no timeout, no circuit breaker, unbounded query |
| GKE | JVM heap > 75% of container limit, missing probe, `latest` image tag in prod |
| Correctness | N+1 query, missing `@Transactional` on multi-write method, swallowed exception |
| Breaking-risk | Public method signature used by 4+ callers, customer API response shape change |

### CodeFix file format — `CodeFixes/FIX-NNN-{short-name}.md`

```markdown
# FIX-{NNN}: {short title}
> Severity: P1 | P2 | P3   Category: SECURITY | RELIABILITY | GKE | CORRECTNESS | BREAKING-RISK
> _Generated: {date}_

## Location
- File: `src/main/java/com/.../UserController.java`
- Line: 47-52
- Method signature: `public ResponseEntity<UserResponse> createUser(@RequestBody UserRequest req)`

## Problem
{1-2 sentences — what is wrong and why it matters}

## Risk if Unhandled
{What breaks: e.g. "Unvalidated input reaches DB layer — injection / bad-data risk.
Customer-facing endpoint, so impacts CustomerA integration."}

## Current Code
```java
@PostMapping("/users")
public ResponseEntity<UserResponse> createUser(@RequestBody UserRequest req) {
    User saved = userService.create(req);
    return ResponseEntity.ok(mapper.toResponse(saved));
}
```

## Suggested Fix
```java
@PostMapping("/users")
public ResponseEntity<UserResponse> createUser(@Valid @RequestBody UserRequest req) {
    User saved = userService.create(req);
    return ResponseEntity.status(HttpStatus.CREATED).body(mapper.toResponse(saved));
}
// + add constraints to UserRequest fields:
//   @NotBlank @Email private String email;
//   @NotNull  private RoleEnum role;
```

## Why This Fix Is Safe
{e.g. "Adds validation only; success/error response shapes unchanged;
GlobalExceptionHandler already returns the {code,message,timestamp} format
for MethodArgumentNotValidException. No customer contract change."}

## Verification
- [ ] Existing tests still pass (`mvn test`)
- [ ] Add test: `createUser_blankEmail_returns400`
- [ ] Confirm error response matches patterns.md format
```

## Rules for Applying Fixes

### CodeFix generation rules

1. **One file per finding** — never combine. Keeps each reviewable + git-diffable.
2. **Always include the real signature and line numbers** — not approximate.
3. **Current code block must be the actual code** copied from the file.
4. **Suggested fix must be minimal** — fix only the issue, no refactoring.
5. **"Why safe" section is mandatory** — explains it won't break contracts.
6. **Never auto-apply** — these are suggestions. The developer/agent applies them deliberately.
7. **Cross-link**: each FIX file referenced in `_INDEX.md`; `_INDEX.md` referenced in `AGENTS.md`.
