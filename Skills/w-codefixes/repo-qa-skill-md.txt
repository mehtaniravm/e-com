# Repo QA Skill

Answers questions from `.claude/context/` files only. Never re-scans source.

---

## File Selection Map

Load **only** the file(s) that match the question:

| Question theme | Load |
|---------------|------|
| Auth, security, Apigee headers | auth.md |
| Endpoints, API surface, entities | apis.md |
| How to write code, what pattern | patterns.md |
| GKE, K8s, memory, probes, scaling | gke.md |
| Dependencies, who calls what, blast radius | deps.md |
| Tests, coverage, how to write tests | tests.md |
| Stack, Java version, Spring Boot version | stack.md |
| "Is it safe to change X" | deps.md + auth.md |
| "How do I add a new endpoint" | patterns.md + auth.md |
| "Story points for X" | apis.md + deps.md |
| General / unclear | CLAUDE.md root (index), then expand |

---

## Answer Rules

1. **Answer in first sentence.** Never bury the lead.
2. **Paste tables verbatim** when the table IS the answer.
3. **Quote exact values** from docs — class names, header names, config keys.
4. **Stale doc warning:** if `_Last generated_` in CLAUDE.md > 90 days → add `⚠️ Docs may be stale. Re-run code-archaeology to refresh.`
5. **Gap in docs:** "Not in context files. To find this: check {specific file in repo}."
6. **Never invent** class names, config values, or endpoint paths.
7. **Source line** at end: `Source: {filename}` — one line, never inline.
8. **No preamble.** Never start with "Great question!" or "Sure, I can help."

## Standard Answer Templates

### Auth question
```
Auth model ({service-name}):

Pattern A — Basic Auth via Apigee:
  Customer sends Basic Auth → Apigee validates → strips header
  Service receives: X-Consumer-ID: {id} (NOT Authorization header)
  Rule: never parse Basic Auth in service code

Pattern B — JWT passthrough:
  Customer sends Bearer JWT → Apigee validates signature → forwards
  Service receives: Authorization: Bearer {token}
  JwtAuthFilter validates locally; extracts: {claims from auth.md}

Public endpoints (no auth): {list from auth.md}

Source: .claude/context/auth.md
```

```
Risk assessment — {class/method/endpoint}:

In deps.md Outbound Calls: {YES — called by: list | NO}
In auth.md Customer Contracts: {YES — consumers: list | NO}
In CLAUDE.md Danger Zones: {YES — reason | NO}

Recommendation:
  HIGH → write characterization test first; get human approval; notify consumers
  MED  → write characterization test first; normal code review
  LOW  → normal code review; add test for change
```

### "How do I add a new endpoint?"
```
Steps for {service-name} (from patterns.md):

1. DTO: create {Entity}Request + {Entity}Response in dto/ package
2. Mapper: MapStruct mapper class in mapper/ (NOT manual setters)
3. Service: add method with @Transactional
4. Controller: add handler — match existing style
5. Auth:
   - BASIC_AUTH_VIA_APIGEE → read X-Consumer-ID (do not parse Basic Auth)
   - JWT → JwtAuthFilter handles; add @PreAuthorize for role check
   - PUBLIC → add path to SecurityConfig permitAll()
6. Validation: @Valid on @RequestBody; constraints on DTO fields
7. Error handling: throw existing exception types → GlobalExceptionHandler catches
8. Tests: unit (extend BaseServiceTest) + controller (@WebMvcTest)
   Add X-Consumer-ID header for Pattern A in controller test

Source: .claude/context/patterns.md + auth.md
```

### "How do I write a test?"
```
Test conventions for {service-name} (from tests.md):

Framework: JUnit {version} — do NOT use other version
Base class: {BaseServiceTest / BaseControllerTest} — always extend

Unit test setup:
  @RunWith(MockitoJUnitRunner.class)  [JUnit 4]
  @ExtendWith(MockitoExtension.class) [JUnit 5]
  @Mock {dependencies}; @InjectMocks {subject under test}

Controller test setup:
  @WebMvcTest({ControllerClass}.class)
  @MockBean {service dependencies}
  Pattern A: mockMvc.perform(get(...).header("X-Consumer-ID","test-id"))
  Pattern B: mockMvc.perform(post(...).header("Authorization","Bearer "+jwt))

Naming: methodName_condition_expectedResult
Example: getUserById_userNotFound_returns404

Source: .claude/context/tests.md + patterns.md
```
