What Changed From the Previous Version
Three things you asked for — now built in
📁 Many Small Files

The old version made 6 big docs. Now: many tiny files under 80 lines each — one concern per file (stack, auth, gke, tests…). Loads faster, fewer tokens, easier to update one piece without touching others.

🤖 Both Tools Wired

Files now land in both tools' native locations: CLAUDE.md + .claude/ for Claude, .github/copilot-instructions.md + .github/prompts/ for Copilot, and a shared AGENTS.md both read.

🔧 CodeFixes Folder

New CodeFixes/ folder — one file per risky finding with exact file path, line number, method signature, current code, and suggested replacement code. Plus a "why this is safe" section.

💡
Why split into many small files? When a developer asks "how does auth work?", the tool loads only auth.md (~40 lines) instead of a 400-line mega-doc. When code changes, you regenerate only the affected file. And Git diffs stay clean — a dependency change touches only deps.md.

📄 Example: CodeFixes/FIX-001-missing-validation.md
# FIX-001: Missing @Valid on createUser
> Severity: P1   Category: SECURITY   Generated: 2026-06-02

## Location
- File: src/main/java/com/co/user/UserController.java
- Line: 47-52
- Signature: public ResponseEntity<UserResponse> createUser(@RequestBody UserRequest req)

## Problem
@RequestBody is not annotated @Valid — no constraint checking on input.

## Risk if Unhandled
Unvalidated input reaches DB layer. Customer-facing endpoint (CustomerA).

## Current Code
public ResponseEntity<UserResponse> createUser(@RequestBody UserRequest req) {
    User saved = userService.create(req);
    return ResponseEntity.ok(mapper.toResponse(saved));
}

## Suggested Fix
public ResponseEntity<UserResponse> createUser(@Valid @RequestBody UserRequest req) {
    User saved = userService.create(req);
    return ResponseEntity.status(HttpStatus.CREATED).body(mapper.toResponse(saved));
}
// + add to UserRequest: @NotBlank @Email String email; @NotNull RoleEnum role;

## Why This Fix Is Safe
Adds validation only. GlobalExceptionHandler already returns {code,message,
timestamp} for MethodArgumentNotValidException. No customer contract change.

## Verification
- [ ] Existing tests pass (mvn test)
- [ ] Add test: createUser_blankEmail_returns400

📐 CodeFix Rules
  • One file per finding — never combined. Each is reviewable + git-diffable.
  • Real signatures + line numbers — never approximate.
  • Minimal fix — fixes only the issue, no refactoring.
  • "Why safe" mandatory — explains no contract breakage.
  • Never auto-applied — suggestions only; dev/agent applies deliberately.
  • Cross-linked — each FIX in _INDEX.md; index in AGENTS.md.
🚨
Breaking-risk fixes need human approval. Any FIX file tagged BREAKING-RISK (signature change, customer response shape) must not be applied by an agent autonomously — it requires human sign-off and possibly customer notice. The skill marks these clearly.
