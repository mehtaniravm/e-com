DETAIL

Login Flow — Step-by-Step

# Action Actor Detail / Security Note
1 Client POSTs credentials Client → Ingress HTTPS only. Body: { email, password }. Rate-limited: 10 attempts/15 min per IP via Bucket4j.
2 Lookup credentials record auth-service Query credentials table by email. Return generic 401 if not found — no user enumeration.
3 Verify password hash auth-service BCrypt.verify(password, storedHash) with cost=12. Constant-time comparison prevents timing attacks.
4 Check account status auth-service If INACTIVE → 423. If LOCKED (brute-force) → 423 with lockout expiry in response.
5 Sign Access Token (RS256) auth-service Claims: sub, iss, aud, iat, exp (+15min), jti (UUIDv7), myapp/roles, myapp/accountStatus. Private key from GCP Secret Manager.
6 Generate Refresh Token auth-service → Redis/DB Opaque 256-bit random UUID. Store SHA-256(token) in DB with userId, deviceHint, expiresAt. Never store raw token.
7 Return tokens auth-service → Client AT in JSON body. RT as Set-Cookie: refresh_token=...; HttpOnly; Secure; SameSite=Strict; Path=/auth/refresh
8 Client stores AT in memory Client (SPA) Never localStorage or sessionStorage. Memory only. RT cookie is inaccessible to JS.

Token Anatomy

What lives inside the JWT — and what never should

ACCESS TOKEN

JWT Structure (RS256 — asymmetric)

JWT = Header . Payload . Signature  |  Algorithm: RS256  |  TTL: 15 minutes
Header
alg"RS256"
typ"JWT"
kid"key-2024-01"
kid identifies which public key in JWKS to use for verification. Enables key rotation without service restart.
Standard Claims (Payload)
sub"usr_01J8KXY..."
iss"https://auth.myapp.io"
aud"myapp-api"
iat1700000000
exp1700000900
jti"tok_01J9ABC..."
jti is the unique token ID — added to Redis blocklist on logout or role change for immediate revocation.
Custom Claims (namespaced)
myapp/roles["ROLE_CUSTOMER"]
myapp/status"ACTIVE"
Custom claims are namespaced with myapp/ to prevent collision with standard registries.
NEVER in JWT:
email, name, phone, address, payment info, passwords, session data, or any PII. JWT payload is base64 — not encrypted.
REFRESH TOKEN

Opaque Token Design

PropertyValue
Format256-bit cryptographically random UUID
Storage (client)HttpOnly Secure SameSite=Strict cookie
Storage (server)SHA-256 hash only — never raw token
TTL7 days
RotationEvery use — old token revoked on refresh
Reuse detectionOld RT reused → revoke ALL tokens for user
ScopePath=/auth/refresh — never sent to other endpoints
KEY MANAGEMENT

RS256 Key Rotation

Private Key

Stored exclusively in GCP Secret Manager. Injected into auth-service pod at startup via K8s secret. Never committed to source control. Rotation: new key pair created, old kid retained in JWKS until all tokens with that kid expire (≤ 15 min).

Public JWKS Endpoint

GET /auth/.well-known/jwks.json — public, unauthenticated. All downstream services cache JWKS locally (TTL = 1 hour). Cache-Control headers set accordingly. On key rotation, downstream services auto-refresh when signature verification fails with cached key.

// Custom policy bean — order ownership check
@Component("orderAccessPolicy")
public class OrderAccessPolicy {
    public boolean isOwner(Authentication auth, String orderId) {
        var order = orderRepository.findById(orderId).orElseThrow();
        return order.getCustomerId().equals(auth.getName()); // sub == userId
    }
}

Token Revocation
& Invalidation

Handling logout, role changes, and account deactivation immediately

FLOW A

Logout

1
Client → POST /auth/logout
Sends Bearer AT in Authorization header. RT cookie sent automatically by browser to Path=/auth/refresh. Both are processed.
2
Add jti to Redis blocklist
Extract jti and exp from AT. SET jti:{jti} 1 EX {exp - now} — TTL matches AT remaining lifetime. No permanent storage needed.
3
Revoke Refresh Token
SHA-256 hash the RT from cookie. UPDATE refresh_tokens SET revoked_at = NOW() WHERE token_hash = ?
4
Clear cookie + return 204
Set-Cookie: refresh_token=; Max-Age=0; HttpOnly; Secure to clear the cookie on the client side.
FLOW B

Role Change / Deactivation

1
Admin calls PATCH /admin/users/{id}/roles
identity-service updates the user_roles table and records the change in audit log.
2
identity-service → auth-service (sync call)
POST /internal/revoke-user/{userId} — internal endpoint, ROLE_SERVICE only. auth-service queries all active refresh_tokens for this userId.
3
Bulk revoke all refresh tokens
UPDATE refresh_tokens SET revoked_at = NOW() WHERE user_id = ? AND revoked_at IS NULL
4
Add jti pattern to Redis blocklist
Store SET user-block:{userId} 1 EX 900 (15 min = max AT TTL). JwtFilter checks this key by sub claim — any AT for this user is rejected within milliseconds.
5
User forced to re-login
Next API call returns 401. RT is revoked so refresh fails too. User must authenticate fresh — new AT will contain updated roles.
EDGE CASES

Refresh Token Reuse Detection (Token Theft)

Attack scenario: An attacker steals a RT (e.g. via network interception). Both the attacker and the legitimate client hold the same RT. The first to use it causes the other to be detected as a potential thief.
EventSystem ResponseOutcome
Legitimate user uses RT → gets new RT Old RT marked revoked_at = NOW(), new RT issued Normal flow
Attacker presents old (revoked) RT RT found in DB but revoked_at IS NOT NULL Reuse detected
Reuse detected Revoke ALL refresh tokens for this userId immediately Both sessions invalidated — user must re-login
User contacts support Admin sees reuse event in audit log with timestamps Potential breach investigation triggered
REDIS KEYS

Blocklist Key Schema

# Individual token revocation (logout)
SET jti:tok_01J9ABC...   "1"   EX 900       # TTL = remaining AT lifetime (max 900s)

# User-level block (role change / deactivation)
SET user-block:usr_01J8K... "1"  EX 900    # blocks ALL tokens for this userId

# JwtFilter check order (fail-fast):
1. Verify RS256 signature           → fail → 401
2. Check exp / iss / aud claims      → fail → 401
3. GET user-block:{sub}              → exists → 401
4. GET jti:{jti}                     → exists → 401
5. Populate SecurityContext          → proceed
6. @PreAuthorize role check           → fail → 403

SPRING SECURITY

SecurityFilterChain + JwtFilter

SecurityConfig.java
@Configuration
@EnableMethodSecurity(prePostEnabled = true)
public class SecurityConfig {

  @Bean
  public SecurityFilterChain filterChain(HttpSecurity http,
                                           JtiBlocklistFilter jtiFilter) throws Exception {
    return http
      .csrf(AbstractHttpConfigurer::disable)            // stateless REST API
      .sessionManagement(s -> s
        .sessionCreationPolicy(STATELESS))
      .authorizeHttpRequests(auth -> auth
        .requestMatchers(
          "/auth/**",
          "/products/**",
          "/actuator/health"
        ).permitAll()
        .requestMatchers("/admin/**").hasAnyRole("ADMIN", "SUPPORT")
        .requestMatchers("/internal/**").hasRole("SERVICE")   // svc-to-svc
        .anyRequest().authenticated())
      .oauth2ResourceServer(rs -> rs
        .jwt(jwt -> jwt
          .jwkSetUri("${auth.jwks-uri}")               // 12-factor: env var
          .jwtAuthenticationConverter(jwtConverter())))
      .addFilterBefore(jtiFilter, BearerTokenAuthenticationFilter.class)
      .exceptionHandling(ex -> ex
        .authenticationEntryPoint(problemDetailEntryPoint())
        .accessDeniedHandler(problemDetailAccessDeniedHandler()))
      .build();
  }

  @Bean
  public JwtAuthenticationConverter jwtConverter() {
    var converter = new JwtAuthenticationConverter();
    var rolesConverter = new JwtGrantedAuthoritiesConverter();
    rolesConverter.setAuthoritiesClaimName("myapp/roles");
    rolesConverter.setAuthorityPrefix("");                // roles already prefixed ROLE_
    converter.setJwtGrantedAuthoritiesConverter(rolesConverter);
    return converter;
  }
}
JtiBlocklistFilter.java
@Component
public class JtiBlocklistFilter extends OncePerRequestFilter {

  private final RedisTemplate<String, String> redis;

  @Override
  protected void doFilterInternal(HttpServletRequest req,
                                      HttpServletResponse res,
                                      FilterChain chain) throws ... {
    var token = extractBearerToken(req);
    if (token != null) {
      var jwt = parseJwtUnsafe(token);                    // parse without verify (done by Spring)
      var sub = jwt.getClaimAsString("sub");
      var jti = jwt.getClaimAsString("jti");

      // Check user-level block first (role change / deactivation)
      if (Boolean.TRUE.equals(redis.hasKey("user-block:" + sub))) {
        sendUnauthorized(res, "Session invalidated");
        return;
      }

      // Check individual token revocation (logout)
      if (Boolean.TRUE.equals(redis.hasKey("jti:" + jti))) {
        sendUnauthorized(res, "Token has been revoked");
        return;
      }
    }
    chain.doFilter(req, res);
  }
}
TOKEN ISSUANCE

Signing Access Tokens

TokenService.java
@Service
public class TokenService {

  private final RSAPrivateKey privateKey;   // injected from GCP Secret Manager
  private final String keyId;               // for JWKS kid rotation
  private final Duration accessTokenTtl = Duration.ofMinutes(15);

  public String issueAccessToken(UserCredential cred, List<String> roles) {
    var now = Instant.now();
    return Jwts.builder()
      .header().keyId(keyId).and()
      .subject(cred.getUserId())
      .issuer("https://auth.myapp.io")
      .audience().add("myapp-api").and()
      .issuedAt(Date.from(now))
      .expiration(Date.from(now.plus(accessTokenTtl)))
      .id(UlidCreator.getMonotonicUlid().toString())    // jti = sortable unique ID
      .claim("myapp/roles", roles)
      .claim("myapp/status", cred.getStatus().name())
      .signWith(privateKey, Jwts.SIG.RS256)
      .compact();
  }

  public String issueRefreshToken(String userId, String deviceHint) {
    var rawToken = generateSecureToken();              // 256-bit random
    var tokenHash = sha256Hex(rawToken);
    refreshTokenRepository.save(RefreshToken.builder()
      .userId(userId)
      .tokenHash(tokenHash)
      .deviceHint(truncate(deviceHint, 100))
      .expiresAt(Instant.now().plus(Duration.ofDays(7)))
      .build());
    return rawToken;                                  // only time raw token is visible
  }
}
